# Dunamis Studios Brand Kit

This directory is the canonical source of truth for Dunamis Studios visual
identity. Anything outside the website that uses the logo — HubSpot
marketplace listings, social profiles, email signatures, press kits, app
icons, deck templates — should pull from here.

Three principles govern every file in this folder:

1. **Consistency.** The mark and wordmark match what renders on
   `dunamisstudios.net` exactly. If the site logo changes, the assets
   here change with it. No drift.
2. **No guesswork.** Every color, font weight, and spacing value is
   documented. Don't infer, don't interpolate, don't "design around" the
   spec — if something feels missing, ask for it to be added rather
   than making it up.
3. **Single source of truth.** SVG files are authoritative. PNGs are
   build artifacts regenerated by `npm run brand:build`. Never hand-edit
   a PNG.

## Logo System

Three logo variants, each with a dark-background version and a light-
background (`-inverse`) version. Six SVGs total.

### Mark only

Square, minimal. A single geometric glyph — a faint outlined circle, a
solid left-arc, and a small accent dot in the upper right. Stylized "δ"
(delta, the Greek root of _dunamis_).

- `logo-mark.svg` — currentColor, drops into any dark or light context
  when the parent element sets `color`.
- `logo-mark-inverse.svg` — literal `#0a0a0a` dark foreground, for
  drop-in use on light backgrounds without CSS.

### Full horizontal

Mark + wordmark side by side, mark on the left. The site header shape.

- `logo-full.svg` — currentColor mark + wordmark. "Studios" at
  50% `fill-opacity` so it tonally steps back from "Dunamis" in any
  color context.
- `logo-full-inverse.svg` — fixed `#0a0a0a` primary + `#737373` muted,
  for light backgrounds.

### Full stacked

Mark on top, wordmark below, both centered. Square-ish proportions for
avatars, app icons, and promotional card corners.

- `logo-full-stacked.svg` — flexible, currentColor.
- `logo-full-stacked-inverse.svg` — light-bg fixed hex.

## Colors

All hex values are the actual literals used in code. The semantic
tokens map to CSS variables in `src/app/globals.css`; external
consumers should use the hex directly since CSS variables don't travel
outside the web app.

### On dark backgrounds (primary brand context)

| Role            | Hex       | CSS variable    | Usage                                |
| --------------- | --------- | --------------- | ------------------------------------ |
| Background      | `#0a0a0a` | `--bg`          | Page bg in dark mode, OG image bg   |
| Foreground      | `#eaeaea` | `--fg` (~=)     | Wordmark "Dunamis", icon strokes     |
| Muted foreground | `#a1a1a1` | `--fg-muted`    | Wordmark " Studios", subtitle copy   |
| Accent indigo   | `#6d5cf5` | `--accent`      | Link/CTA color, Stripe Elements primary |
| Accent light    | `#a89bff` | `--color-brand-400` | Italic flourishes, OG image highlight |

### On light backgrounds (inverse)

| Role            | Hex       | Usage                             |
| --------------- | --------- | --------------------------------- |
| Background      | `#ffffff` | Default light bg                  |
| Foreground      | `#0a0a0a` | Wordmark "Dunamis", icon strokes  |
| Muted foreground | `#737373` | Wordmark " Studios"               |

Do not use the accent indigo as the primary logo color in either
context — it's reserved for interactive affordances (buttons, links,
highlights), not identity.

## Typography

The wordmark renders in **Fraunces**, a variable serif display typeface
by Phaedra Charles, Pooja Saxena, and Nick Sherman (SIL Open Font
License). Shipped via `next/font/google` in `src/app/layout.tsx` and
available here via the `@fontsource/fraunces` npm package.

| Element             | Font     | Weight | Style  | Size (in header) | Tracking    |
| ------------------- | -------- | ------ | ------ | ---------------- | ----------- |
| "Dunamis"           | Fraunces | 500    | Normal | 17px             | −0.02em     |
| "Studios"           | Fraunces | 500    | Normal | 17px             | −0.02em     |
| Headlines (display) | Fraunces | 500    | Normal | 32–96px depending on breakpoint | −0.02 to −0.035em |

**Both words of the wordmark use the same weight and style.** The
tonal differential between them is color only: "Dunamis" in the
primary foreground, " Studios" in the muted foreground (50%
`fill-opacity` in flexible SVGs, literal `#737373` in inverse SVGs).

Body text on the website uses **Geist** (sans); technical detail
(portal IDs, timestamps) uses **Geist Mono**. Those fonts aren't part
of the identity and don't appear in any brand file here.

## Spacing Rules

**Clear space.** Leave empty space around the logo equal to the height
of the mark on every side. If the mark renders at 32px, no other
content should appear within 32px of the logo's bounding box. For the
horizontal lockup, the clear space is measured from the outermost
stroke of the mark, not the text. For the stacked lockup, measure from
the union bounding box.

**Don't.**

- Recolor the wordmark outside the provided dark / light variants.
- Italicize, condense, or substitute the font in the wordmark.
- Rotate the mark or the lockup.
- Apply drop shadows, outlines, glows, or 3D effects.
- Place the dark-variant logo on a background below 40% luminance
  contrast. Use the inverse variant instead.
- Re-create the mark with different line weights, different arc sweep,
  or a different accent-dot position. The geometry is the identity.

**Do.**

- Scale the logo uniformly (never distort aspect).
- Use the full color of the underlying page / surface as the background
  and let the logo sit on it.
- Swap between variants as the background luminance demands.
- Use the PNGs when embedding into contexts that don't render SVG
  reliably (HubSpot rich text, email clients, certain social platforms).

All PNGs regenerate via `npm run brand:build`. Don't edit PNGs
directly — edit the SVG and rebuild.

### Source SVGs

| File                               | Variant           | Background context |
| ---------------------------------- | ----------------- | ------------------ |
| `logo-mark.svg`                    | Mark only         | Dark (flex)        |
| `logo-mark-inverse.svg`            | Mark only         | Light              |
| `logo-full.svg`                    | Horizontal lockup | Dark (flex)        |
| `logo-full-inverse.svg`            | Horizontal lockup | Light              |
| `logo-full-stacked.svg`            | Stacked lockup    | Dark (flex)        |
| `logo-full-stacked-inverse.svg`    | Stacked lockup    | Light              |

### PNG exports

Each SVG rasterizes to 7 PNG sizes under `brand/png/` with the naming
pattern `{stem}-{size}.png`:

| Size   | Typical use                                           |
| ------ | ----------------------------------------------------- |
| 16px   | Browser favicons (mark only; horizontal/stacked illegible here) |
| 32px   | Tab-bar icons, OS notification icons                  |
| 64px   | Standard web iconography, small UI elements           |
| 128px  | Retina favicons, small app icons                      |
| 256px  | HubSpot marketplace listing mini-icon, standard app icon |
| 512px  | HubSpot marketplace hero icon, PWA install icon       |
| 1024px | Apple App Store, high-resolution print, hero visuals  |

Non-square variants (`logo-full-*`, `logo-full-stacked-*`) preserve
the SVG aspect ratio — the `size` value represents the target **width**
in pixels, and height is derived from the viewBox. `logo-full-256.png`
is 256×47; `logo-full-stacked-256.png` is 256×179.

## Regenerating Assets

```bash
npm run brand:build
```

Reads every SVG in `brand/`, rasterizes each to 7 PNG sizes, and writes
the output under `brand/png/`. Uses `@resvg/resvg-js` with
`@fontsource/fraunces` woff2 buffers — deterministic output regardless
of which machine runs the build, no system-level font install required.

If you change an SVG, commit the SVG + the regenerated PNGs together
so the checked-in brand/ directory stays internally consistent.
